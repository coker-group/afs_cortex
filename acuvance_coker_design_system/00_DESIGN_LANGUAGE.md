## 2. Logo

### What the Logo Is

The Acuvance Coker logo is a **dual-mark composite** consisting of three elements:

1. **"ACUVANCE"** — geometric sans-serif letterforms in Dark Charcoal (`#22272D`), all caps, with a distinctive curved crossbar on the "A"
2. **Vertical divider bar** — a thin vertical rule in mid-gray (`#C5C9CB`)
3. **"Coker"** — the classic hand-lettered script wordmark in Deep Blue (`#0C36B4`) with registered trademark symbol (®)

These three elements form a single indivisible unit. The logo **cannot be reproduced using any system font** — it must always be embedded as an **image asset** (PNG or SVG).

### Logo Variants

| Variant | File | Use On |
|---|---|---|
| **Full color** | `acuvance-coker-logo-color.png` | White or light gray backgrounds |
| **White (reverse)** | `acuvance-coker-logo-white.png` | Dark, blue, or photographic backgrounds |
| **Coker script only** (white SVG) | `coker-script-white.svg` | Special contexts where only the script mark is needed (rare — prefer full logo) |

### Logo Rules — Non-Negotiable

| Rule | Detail |
|---|---|
| **Always an image** | Never attempt to render the logo using CSS text, font styling, or Unicode characters |
| **Never modify** | No stretching, skewing, recoloring, cropping, or adding effects (shadow, glow, outline) |
| **Clear space** | Maintain clear space equal to the height of the "A" in ACUVANCE on all four sides |
| **Minimum width** | 1.5 inches (print/PPTX) or 180px (screen) — the dual-mark needs room to breathe |
| **No partial use** | Never display "ACUVANCE" without "Coker" or vice versa in logo contexts (the script-only variant is for legacy/transitional use) |
| **Vertical divider** | The pipe separator between ACUVANCE and Coker is part of the mark — never remove or replace it |

### Logo Color Rules

| Background | Logo Version |
|---|---|
| White / Light Gray (`#F3F4F5`) / Light Blue tint | Full color (dark ACUVANCE + blue Coker script) |
| Deep Blue / Vibrant Blue / Dark Charcoal | White reverse version |
| Photography (with overlay) | White reverse version |
| Mid-tone backgrounds | Evaluate contrast — prefer white reverse if background is darker than `#999999` |

### HTML Implementation

```html

  src="assets/acuvance-coker-logo-color.png"
  alt="Acuvance Coker"
  style="height: 36px; width: auto; display: block;"

<!-- NEVER DO THIS — the logo is not reproducible as text -->

```

### Placeholder When Asset Is Unavailable


### Accent Tint Progressions

**Amber:** `#F4B73F` → `#F6C565` → `#F8D48C` → `#FBE2B2` → `#FDF1D9`
**Teal:** `#4CC6C6` → `#70D1D1` → `#94DDDD` → `#B7E8E8` → `#DBF4F4`
**Green:** `#169873` → `#45AD8F` → `#73C1AB` → `#A2D6C7` → `#D0EAE3`

### Color Usage Rules

1. **Vibrant Blue (`#004BFF`) dominates.** It is the signature color — use it for slide backgrounds, color-block shapes, table headers, primary CTAs, and first data series.
2. **Deep Blue (`#0C36B4`) supports.** Use for headings in documents, table borders, and logo rendering.
3. **Never use default Office colors.** No default reds, greens, oranges, or generic blues not defined above.
4. **White text on blue backgrounds. Dark text on light backgrounds.** Never place dark text on dark or light text on light.
5. **Accent colors require justification.** Amber, teal, and green are for data differentiation — never for decoration.
6. **Semantic colors are fixed.** Green = favorable/positive. Red = unfavorable/negative. Never reverse these meanings.

## 4. Typography

### The Two-Font System

Acuvance Coker maintains two typographic environments:

| Font | Environment | When |
|---|---|---|
| **Acid Grotesk** (FFF Acid Grotesk family) | Brand & marketing only | Website, brand campaigns, design-team-produced marketing materials |
| **Arial** | Client deliverables + all AI output | Every report, slide deck, dashboard, app, or document generated for or shared with clients |

**AI-generated deliverables always use Arial.** Never import, reference, or use Acid Grotesk in any file produced by this design system.

### Font Stack

```css
font-family: Arial, Helvetica, sans-serif;
```

### Permitted Font Weights

| Weight | Value | Use |
|---|---|---|
| Regular | 400 | Body text, table cells, paragraphs |
| Semibold | 600 | Section labels, card titles, badges |
| Bold | 700 | Slide titles, table headers, H1–H2 headings |

No other weights (100, 200, 300, 500, 800, 900) are permitted.

### Type Scale

| Element | Size (rem) | Size (px) | Weight | Line Height |
|---|---|---|---|---|
| H1 — report/cover title | 1.875 | 30 | 700 | 1.2 |
| H2 — slide/section title | 1.5 | 24 | 700 | 1.2 |
| H3 — section label | 1.25 | 20 | 600 | 1.2 |
| H4 — card title | 1.125 | 18 | 600 | 1.3 |
| Body text | 1.0 | 16 | 400 | 1.5 |
| Table body | 0.875 | 14 | 400 | 1.5 |
| Table header | 0.875 | 14 | 700 | 1.5 |
| Caption / footnote | 0.75 | 12 | 400 | 1.4 |
| Badge / label | 0.75 | 12 | 600 | 1.2 |

### Typography Rules

- **No italic** in data visualizations or tables. Italic is permitted only in footnotes, source citations, and the brand tagline.
- **No underline** except for hyperlinks.
- **ALL CAPS** only for badges, labels, and the "ACUVANCE" portion of the logo.
- **Title case** for slide titles and section headers.
- **Sentence case** for body text and callouts.

## 5. Spacing & Layout Philosophy

### Spacing Scale (4px base)

| Token | Value |
|---|---|
| `--sp-1` | 4px |
| `--sp-2` | 8px |
| `--sp-3` | 12px |
| `--sp-4` | 16px |
| `--sp-5` | 20px |
| `--sp-6` | 24px |
| `--sp-8` | 32px |
| `--sp-10` | 40px |
| `--sp-12` | 48px |
| `--sp-16` | 64px |

### Layout Principles

1. **Generous white space.** Acuvance Coker's design language is clean and confident. Space communicates authority — do not crowd content.
2. **Consistent margins.** Minimum 0.5" in print, `--sp-6` (24px) in screen contexts.
3. **Alignment over decoration.** Use grid alignment and consistent spacing to create visual order — do not rely on decorative elements to organize content.
4. **Content density scales with context:** Slides are sparse (one idea per slide). Dashboards are denser (multiple KPIs). Documents fall between.
5. **No orphaned elements.** Never leave a single bullet, one table row, or a lone stat card on a slide — rebalance.

### Border & Radius System

| Token | Value | Use |
|---|---|---|
| `--radius-sm` | 4px | Buttons, badges, small cards |
| `--radius-md` | 8px | Cards, containers |
| `--radius-lg` | 12px | Modals, feature panels |
| `--radius-pill` | 999px | Pills, tags, toggle switches |

### Shadow System

| Token | Value | Use |
|---|---|---|
| `--shadow-sm` | `0 1px 4px rgba(0,0,0,0.06)` | Subtle lift — filter chips, tags |
| `--shadow-card` | `0 2px 8px rgba(0,0,0,0.08)` | Cards, panels |
| `--shadow-elevated` | `0 4px 16px rgba(0,0,0,0.12)` | Modals, dropdowns, popovers |

## 6. Graphic Identity Elements

### Grid Shapes

Grid Shapes are Acuvance Coker's **primary decorative device** — three stacked, offset rectangles forming a layered corner accent. They are carried forward from the Coker brand.

**Rules:**
- Corner placement only — never centered or floating
- 90-degree rotations only (0°, 90°, 180°, 270°)
- Always three layers with progressive offset
- Blue family colors only (`--ac-blue`, `--ac-deep-blue`, `--ac-blue-20`)
- Never cover content — decorative background only
- 15–25% opacity on light backgrounds; 20–35% on dark

**When to use:** Cover slides, section dividers, dashboard headers. **Never** on data content slides or inside charts.

### Radial Gradient

```css
background: radial-gradient(circle at top left, #004BFF 0%, #0C36B4 100%);
```

- Origin point at a corner or edge — never centered
- `#004BFF` at origin → `#0C36B4` radiating outward (never reversed)
- Dark backgrounds only — never on light/white backgrounds
- Maximum one per deliverable
- Never behind data content (charts, tables, KPIs)

### Icon Style

Acuvance Coker presentations use **Microsoft Office icon library** icons in two styles:
- **Outline icons** — for light backgrounds, using Vibrant Blue (`#004BFF`) or Deep Blue (`#0C36B4`)
- **Solid fill icons** — for colored backgrounds, using white or the blue family

Icons are always placed inside **colored circles** or used as standalone accents next to section headers. Never use raw unframed icons floating in content.

## 7. Photography & Imagery

### Core Principles

| Principle | In Practice |
|---|---|
| Real people, real settings | Healthcare professionals in actual clinical or administrative environments |
| Warm but professional | Natural light, confident and engaged expressions |
| No stock clichés | Avoid handshake photos, generic "diverse team smiling" compositions |
| Action over pose | Physicians reviewing data, administrators in meetings, clinical staff in care |
| Compositional space | Leave negative space for text overlay |

### Color Tone

Photography should feel **cool-to-neutral** — blues, grays, whites, navy in the environment. Avoid heavy warm tones (orange/red casts), cluttered backgrounds, or Instagram-style filters.

### Usage

- **Cover slides and section dividers only** — never on data content slides
- Mix full-color and black-and-white within the same deck for visual contrast
- B&W photos work especially well as backgrounds for section dividers
- When photography is unavailable, use the radial gradient cover as fallback

## 8. Voice & Tone

### The Approachable Expert

Acuvance Coker's voice is **authoritative yet accessible** — the voice of a senior advisor who explains complex healthcare matters clearly without condescension.

### Writing Rules

| Rule | Correct | Incorrect |
|---|---|---|
| No contractions | "cannot," "does not" | "can't," "doesn't" |
| Spell out "percent" in prose | "15 percent" | "15%" (prose only — % is fine in charts) |
| Spell out 1–9 | "three hospitals" | "3 hospitals" |
| Company name | "Acuvance Coker" | "Coker," "The Acuvance Coker" |
| Client reference | "the Practice," "the Client" | "you," "your" |
| Compound adjectives | "full-time," "on-site" | "full time," "on site" |
| Phases | "Phase One" | "Phase 1" |

### Slide Titles

Use **insight titles** (what the data means) rather than topic titles (what the slide contains):
- **Insight:** "Ambulatory Volume Recovery Outpaces Inpatient by 12 Percent"
- **Topic:** "Volume Trends Overview"

## 9. Accessibility

All deliverables must meet WCAG 2.1 AA standards:

| Requirement | Standard |
|---|---|
| Text contrast | 4.5:1 minimum against background |
| Large text contrast | 3:1 minimum (18px+ bold or 24px+ regular) |
| Non-text contrast | 3:1 for UI components, chart elements |
| Color not sole indicator | Always pair color with pattern, label, or icon |
| Alt text | All images, charts, and logos must have descriptive alt text |
| Focus indicators | All interactive elements must have visible focus states |
